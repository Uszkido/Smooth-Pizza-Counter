Pizza & Ice Cream Club
Official System & Hardware Documentation

01. Core Engine Architecture

The Pizza & Ice Cream Club AI is powered by an ultra-fast threaded internal tracking engine with support for both IP Cameras (RTSP) and USB Cameras. To combat the pervasive latency issues common with standard OpenCV implementations, entirely dedicated background TCP/FFMPEG extraction subroutines have been built. This ensures the live dashboard is consistently viewing the absolute present millisecond in your kitchen operations.

🔑 Staff Portal Login

The system is secured by a role-based access control system. Use the following default credentials to log in:

  • Manager: admin / CHANGE_ME
  • Viewer: viewer / CHANGE_ME

Passwords can be updated in the Settings page by the Manager.

02. Hardware Requirements & IP Cameras

For flawless enterprise usage during heavy volume shifts, the hardware choices implemented on the shop floor make a dramatic difference. Deep-learning pipeline resolution thresholds are intentionally hard-coded down to imgsz=480 to allow standard CPU deployment without requiring extreme local GPU hardware.

⚠️ Critical Recommendation: 2MP Over 4MP

A staggering amount of processing overhead is intrinsically wasted internally decoding 4 Megapixel (1440p+) streams before the AI even touches the image. We strongly advise installing a 2 Megapixel (1080p) IP Camera. A 2MP stream will absolutely guarantee liquid-smooth zero-lag streaming metrics while preserving 100% computational integrity for the YOLO engine tracking pizzas.

03. Dashboards & Analytics Suite

The application launches natively on port 8040 (by default) bound to 0.0.0.0 utilizing the FastAPI framework. This LAN configuration allows any device on the network to access your command center.

  • Live Dashboard: Displays the dynamically filling Daily Goal Progress Ring. The ring fills as you hit your targeted pizza count, calculating percentage live natively through real-time math.
  • Historical Calendar (Analytics): Your historical data is permanently stored chronologically within your backend via pizza_logs.json. You may click through any date in the calendar and Chart.js will physically map out an exact graphical bar-chart demonstrating exactly which peak hours were slammed.
  • Remote Link Generation: Tapped via a unified batch script. Access through the dashboard pulls the Ngrok tunnel directly, creating an internet URL to monitor from any cellphone world-wide instantly.

04. Calibration Interface

To calibrate the software to a new oven location across franchises:

  • Camera Source Setup: Navigate to the Calibration page, select the specific camera brand (Hikvision, Dahua, Tiandy, Generic), and submit the discrete static internal IP along with Username & Password.
  • Visual Line Sync: Slide the Y-Axis Calibration bar to dynamically test boundaries overlaid right over the live streaming RTSP proxy video feed. Any crossing of a moving pizza bottom trajectory bounds will immediately trigger screen flashes.

05. Shift Reports & Notification Engine

The system constantly polls production timing and automatically handles shift reporting.

🚨 30-Minute Check: If the AI is active but registers 0 pizzas in a 30-minute window, a visual alert triggers ⚠️ IDLE DOWNTIME and an audio beep sounds. A push notification is sent to WhatsApp and Telegram.

📊 Shift Summaries: Managers can send a "Shift Summary" report at any time via the dashboard. At 11:00 PM, a final nightly report is auto-broadcasted showing hourly trends and peaks.
