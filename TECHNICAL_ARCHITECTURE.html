The OpenCV Bridge

We use OpenCV (Open Source Computer Vision Library) to capture frames from RTSP (Hikvision/Dahua) and USB streams. A dedicated background thread extracts frames at 25 FPS, ensuring the AI always looks at the "present" with zero lag.

01

The YOLOv8 Detection Model

At the center of the intelligence is YOLOv8 (You Only Look Once). This convolutional neural network was trained on thousands of variations of oven-baked pizzas to identify them by their specific textures, shapes, and geometry.

Training Process: The model was taught using annotated datasets that include varying lighting conditions (bright kitchens vs dim evenings), ensuring 95%+ accuracy across different store locations.

02

Counting Trajectory Logic

Instead of simple detection, the system uses a Tracking Logic. Each object is assigned a unique ID. We calculate the trajectory of these IDs. Only when a "Pizza" ID crosses our virtual Counting Line (Y-Coordinate) from top to bottom is a count incremented.

This prevents double-counting if a pizza is adjusted near the mouth of the oven.

Persistent JSON Database

To avoid the overhead of heavy SQL databases, we use a structured JSON-based Data Store. This ensures portability and blazing fast reads for the Analytics dashboard. Production data is stored hourly to allow shift-based performance mapping.

Modern PWA Frontend

The interface is built with HTML5, CSS3 (Vanilla), and ES6+ JavaScript. We use Chart.js for real-time data visualization. The entire dashboard is designed as a Progressive Web App (PWA), meaning it can be "installed" on a tablet or mobile phone for one-tap access by the store manager.

The Integrated Process Flow
