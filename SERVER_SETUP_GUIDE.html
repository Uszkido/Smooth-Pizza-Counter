INTRO How PizzaClub Runs as a Server

PizzaClub AI VMS operates as a lightweight local web server built on FastAPI + Uvicorn. When running, it binds to a port on the machine and serves the live dashboard, video stream, and REST API to any browser on the network — exactly like a real server application.

This guide configures PizzaClub to behave like production server software:

  • Starts automatically when Windows boots — no one needs to log in
  • Runs hidden in the background — no visible window consuming screen space
  • Auto-recovers from crashes — watchdog restarts it within 10 seconds
  • Accessible from any device on the local network via LAN IP
  • Remote access via ngrok for monitoring from anywhere in the world

03 🚀 Autostart on Windows Boot

The installer registers PizzaClub as a Windows Task Scheduler task. This is the most reliable way to run background software on Windows — it's the same mechanism used by antivirus software, backup agents, and system utilities.

How it works

  • Trigger: On system startup (30-second delay to let Windows settle)
  • Runs as: SYSTEM account — starts even before anyone logs in
  • Priority: Highest — not throttled by Windows resource governor
  • Hidden: Task launches START_HIDDEN.vbs → WATCHDOG.bat → PizzaClub.exe, all invisible, no CMD window anywhere
  • Closing any terminal: zero effect — the process is owned by SYSTEM, not your user session

Verifying the task manually

1
Open Task Scheduler Press Win + R, type taskschd.msc, press Enter.
2
Find the task In the left panel click Task Scheduler Library. Look for PizzaClub AI Server in the list.
3
Check status The Status column should show Ready. Right-click → Run to test it manually.

Disabling autostart temporarily

In Task Scheduler, right-click PizzaClub AI Server → Disable. The task entry is kept but won't run on next boot. To re-enable: right-click → Enable.
Or run UNINSTALL_SERVICE.bat to remove it completely.

Alternative: Startup Folder (simpler, user-only)

For a quicker setup that only runs when you are logged in (not as SYSTEM): 

1. Press Win+R, type:
shell:startup
2. Create a shortcut to PizzaClub.exe in that folder.
   It will launch every time you log in.
💡 Task Scheduler vs Startup Folder
Method Starts without login Admin required Crash recovery
Task Scheduler (SYSTEM) Yes ✅ Yes (install only) No (add Watchdog)
Startup Folder No ❌ No No
Task Scheduler + Watchdog Yes ✅ Yes (install only) Yes ✅

04 🔄 Crash Recovery — Watchdog

WATCHDOG.bat is a production-grade keep-alive script. It launches PizzaClub.exe, waits for it to exit (for any reason — crash, update, manual close), then restarts it after 10 seconds. All events are timestamped and logged to watchdog.log.

Setting up the Watchdog as the autostart target

For maximum resilience, configure the Task Scheduler task to run the Watchdog instead of PizzaClub.exe directly (so even if the main exe crashes repeatedly, the watchdog keeps trying):

1
Open Task Scheduler → find PizzaClub AI Server → double-click.
2
Go to the "Actions" tab → double-click the existing action.
3
Change Program/Script to: 
C:\PizzaClub\PizzaClub_v2.1_Enterprise\WATCHDOG.bat
Click OK → OK.

Reading the Watchdog Log

Open watchdog.log (in the same folder) in Notepad at any time:

watchdog.log[2026-04-25 08:31:02] PizzaClub Watchdog Started
[2026-04-25 08:31:02] EXE : C:\PizzaClub\PizzaClub.exe
[2026-04-25 08:31:02] Port: 8040
[2026-04-25 08:31:03] Starting PizzaClub.exe...
[2026-04-25 14:55:11] PizzaClub.exe exited. Restarting in 10 seconds...
[2026-04-25 14:55:21] Starting PizzaClub.exe...
✅ Log rotation is automatic
When watchdog.log exceeds 512 KB it is automatically backed up to watchdog.log.bak and a fresh log starts. This prevents unbounded disk growth during long-term operation.

05 🌐 Network Access Setup

Accessing from the same machine

http://127.0.0.1:8040    # always works on the host PC

Accessing from other devices on the same network (LAN)

  1. Find the server machine's IP: open STATUS.bat and read the LAN IP Access line.
  2. On any phone, tablet, or PC on the same WiFi/Ethernet, open: http://<server-ip>:8040
  3. Log in with the manager or viewer credentials.
⚠️ Use a static IP on the server machine
If the server's IP changes (e.g. router reassigns via DHCP), remote devices will lose access. Set a static/reserved IP in your router's DHCP settings for the server's MAC address. Typical home routers have this under "Address Reservation" or "Static DHCP".

Finding your machine's IP address

# Open Command Prompt (Win+R → cmd) and type:
ipconfig
# Look for: "IPv4 Address . . . : 192.168.x.x"

Remote Access over the Internet (ngrok)

ngrok creates a secure temporary URL that tunnels through NAT/firewalls — no router port-forwarding needed. It is included in the package as ngrok.exe.

1
Double-click START_REMOTE_ACCESS.bat It reads the port from config.json automatically.
2
Look for the Forwarding URL in the terminal It will look like: https://abc123.ngrok-free.app
3
Share that URL with anyone — they can access the live dashboard from any device, anywhere in the world. The link is active as long as the script is running.

Permanent Remote Access (ngrok free account)

Free ngrok sessions expire after a few hours. For a stable permanent URL:

  1. Create a free account at ngrok.com
  2. Get your auth token from the dashboard
  3. Run in Command Prompt: ngrok.exe authtoken YOUR_TOKEN
  4. Your sessions will last much longer and you can get a fixed subdomain on paid plans

Windows Firewall

The INSTALL_SERVICE.bat script automatically adds a firewall rule for the configured port. If you need to add it manually: 

# Run in Command Prompt as Administrator:
netsh advfirewall firewall add rule ^
    name="PizzaClub AI Server" ^
    dir=in action=allow protocol=TCP ^
    localport=8040

06 🛠️ Advanced Configuration

Running Multiple Branches from One PC

Copy the entire application folder to a second location (e.g. C:\PizzaClub\Branch2\), give it a different "app_port" in config.json, then run INSTALL_SERVICE.bat in that folder. Name the task something unique when prompted, or edit the task name inside the bat before running. 

Branch 1 — Old Airport:  config.json → "app_port": 8040
Branch 2 — New Airport:  config.json → "app_port": 8050

Changing the Port After Installation

  1. Edit "app_port" in config.json
  2. Run UNINSTALL_SERVICE.bat (as Admin) to remove the old firewall rule
  3. Run INSTALL_SERVICE.bat (as Admin) to register the new port and firewall rule

Changing Passwords

Edit the "users" section in config.json. Changes take effect on the next login — active sessions are not terminated automatically. 

"users": {

07 ❓ Frequently Asked Questions

The server doesn't start automatically after reboot

Run STATUS.bat to confirm the autostart task is [REGISTERED]. If not, re-run INSTALL_SERVICE.bat as Administrator. Also check that the path to PizzaClub.exe hasn't changed since installation.

Other devices on the network can't connect

  1. Confirm STATUS.bat shows firewall [OPEN] ✅ — if not, run INSTALL_SERVICE.bat again
  2. Make sure you're using the server machine's actual LAN IP (not 127.0.0.1)
  3. Try temporarily disabling Windows Defender Firewall to test if it's a firewall issue
  4. Confirm both devices are on the same network — a phone on mobile data won't see LAN devices

How do I check if the server is actually running?

Run STATUS.bat — it checks process, port, firewall, and task registration in one view. Or open http://127.0.0.1:8040 in a browser.

The Watchdog keeps restarting — what does that mean?

Check watchdog.log for timestamps. If restarts happen every few seconds, open pizza_crash_log.txt to see the Python exception that is causing the crash. Common causes: camera IP unreachable at startup (the app retries automatically), or corrupt config.json (re-enter your settings manually).

Can I run PizzaClub on a headless server (no monitor)?

Yes. The app runs entirely in the background. Access the dashboard from any other machine on the network using the server's LAN IP and port. No monitor, keyboard, or mouse needed on the server machine once the service is installed.

How do I completely remove PizzaClub?

  1. Run UNINSTALL_SERVICE.bat as Administrator
  2. Delete the entire application folder
  3. Your browser history may still have the URL cached — this is harmless

Can the server URL be used by a phone app or third-party tool?

Yes. The REST API is available at http://<ip>:<port>/api/stats (returns JSON). Any app that can make HTTP requests can read live pizza counts, hourly data, and branch info. Authentication required: send a valid session cookie from /login.
