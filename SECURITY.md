# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (`main`) | ✅ Yes |
| Older branches | ❌ No — please update to `main` |

---

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

If you discover a security issue in Smooth Pizza Counter, please report it privately:

**Email:** security@vexelinnovations.com  
**Subject line:** `[SECURITY] Smooth Pizza Counter — <brief description>`

Include in your report:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigation (optional)

---

## Response Timeline

| Step | Target time |
|------|------------|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix or mitigation | Dependent on severity |
| Disclosure | Coordinated with reporter |

We take all reports seriously. Critical vulnerabilities will be patched as a priority and a fix released before any public disclosure.

---

## Scope

This policy covers:
- The Flask web application and authentication system
- The detection pipeline and model inference code
- Configuration handling and secrets management

Out of scope: third-party libraries (report those to their respective maintainers).
